How to use the Notes system

Personal reference for publishing standalone HTML pages on garys-page.

How it works

  • Each note is a plain HTML file in public/notes/.
  • The site lists every *.html file except ones that contain <!-- draft --> or <!-- hidden -->.
  • Every note page shows a fixed ← Notes button (via notes-nav.js).
  • Title on the list comes from the file’s <title> (or first <h1>).
  • Date on the list is the file’s last-modified time.
  • Live URL: /notes/slot-01 → serves /notes/slot-01.html.

Publish a note

  1. Pick an empty slot, e.g. public/notes/slot-05.html.
  2. On GitHub: edit the file and replace the whole contents with your HTML.
  3. Remove the line <!-- draft -->.
  4. Set <title>Your title here</title>.
  5. Commit → deploy. It appears on /notes automatically.

Choose which notes show on /notes

Open /notes/manage in the browser — check the notes you want, copy the JSON, and save it as notes-manifest.json in the repo root on GitHub. Only slugs listed under show appear on the public list (in that order).

{
  "show": ["slot-08", "slot-05", "slot-01"]
}

Delete notes-manifest.json or set show to [] to fall back to all non-hidden notes.

Hide a note from the list

Add either marker anywhere in the file to delist without deleting:

<!-- draft -->
<!-- hidden -->

Or simply omit it from notes-manifest.json. Direct URL still works.

Reserved slots

slot-01.html … slot-100.html are pre-created drafts. Use the next free slot or generate more locally:

pnpm generate-slots              # next 10 slots after the highest number
pnpm generate-slots --count 50   # 50 new slots
pnpm generate-slots --help

Images & assets

Put files under public/notes/assets/ and reference them from your HTML:

<img src="/notes/assets/my-diagram.png" alt="..." />

Minimal publish template

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>My note title</title>
</head>
<body>
  <h1>My note title</h1>
  <p>Your content here.</p>
</body>
</html>

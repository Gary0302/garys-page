The MLP Guider Hypothesis: A Unified Geometric Framework for Transformer Reasoning

Semantic manifolds, attention navigation, the nature of reasoning, and the geometric effect of RL — a complete framework with experimental validation from Exp A and Exp-LL

I. Definitions

The following terms appear throughout this report and are defined from first principles.

  • Transformer: The core architecture of mainstream large language models (GPT, Claude, Qwen, etc.). Composed of stacked layers, each containing an Attention mechanism and an MLP.
  • Residual Stream: The accumulated vector representation of a token as it passes through all layers. Each layer's output is added on top of the previous one, not replacing it.
  • Hidden State: The current state of the residual stream at a given layer — a high-dimensional vector (2048 dimensions for Qwen2.5-3B) encoding the model's current understanding of the token.
  • MLP (Multi-Layer Perceptron): One of two sub-modules in each Transformer layer (the other being Attention). It receives the current hidden state and outputs an update vector added back to the residual stream. The central object of this hypothesis.
  • Attention: The mechanism that lets each token attend to other tokens in the sequence, weighting and aggregating their information based on relevance.
  • MoE (Mixture of Experts): A sparse Transformer architecture where the MLP is split into many expert networks, with only a few activated per token. A router decides which experts are activated.
  • lm_head (Language Model Head): The linear projection matrix after the final layer, mapping the hidden state to logits over the vocabulary. Each row corresponds to a token's direction vector in hidden state space.
  • Logit: The raw score the model assigns to each token before conversion to a probability.
  • Cosine Similarity: Measures how close two vectors are in direction, ranging from $-1$ (opposite) to $+1$ (identical direction), with $0$ meaning perpendicular.
  • Semantic Manifold: The geometric principle that meaningful semantic information occupies a low-dimensional surface (manifold) within the high-dimensional hidden state space. Learning is understood as learning to navigate this manifold.
  • Attractor Basin: Borrowed from physics: a region in the loss landscape where gradient flow converges to a local minimum. Correct answers and incorrect answers can each form separate attractors.
  • Saddle Point: A point that is a local maximum in one direction and a local minimum in another. In this hypothesis: a position along the residual stream trajectory where curvature is near zero and the system is most sensitive to the next step taken.
  • Logit Lens: An analysis technique that projects the hidden state at each layer through lm_head to observe what the model would output if it stopped at that layer.
  • RL (Reinforcement Learning): A training method that scores model outputs and adjusts parameters to improve those scores. Includes RLHF, GRPO, and similar approaches.

II. The Core Hypothesis Framework

2.1 Foundation: The Semantic Manifold Hypothesis

All LLMs are learning to operate on low-dimensional semantic manifolds embedded in high-dimensional space. Although a hidden state is a 2048-dimensional vector, meaningful semantic information lives on a lower-dimensional surface. Every layer moves along this surface, trying to reach the region where the correct answer resides. Capability is not about larger dimensionality — it is about learning more precise paths along the manifold.

2.2 Navigation Layer: Attention ≈ MoE Router (Abstract Equivalence)

At the abstract level, Attention and MoE Router do the same thing: use a query to find the most relevant keys in some space, then aggregate the corresponding values. The difference is only in granularity and softness:

Attention
Soft navigation in the token dimension. The current token's hidden state serves as a query; it finds relevance across other tokens (keys) and fuses their information (values). Continuous weighting — all tokens participate.

MoE Router
Hard navigation in the expert dimension. The current token's hidden state determines which experts to activate. Discrete top-k selection — most experts are bypassed.

2.3 MLP as a Trained Guider

This is the core of the hypothesis. The update equation for each Transformer layer is:

$$h_{\text{out}} = h_{\text{in}} + \text{Attention}(h_{\text{in}}) + \text{MLP}(h_{\text{in}})$$

Isolating the MLP's contribution:

$$\Delta h = \text{MLP}(h_{\text{in}})$$

This $\Delta h$ is the MLP's pure contribution vector — it has both a direction and a magnitude. The hypothesis claims: the direction of $\Delta h$ geometrically points toward the embedding direction of the correct answer. Measured with cosine similarity:

$$\text{gap} = \cos\text{sim}(\Delta h,\, v_{\text{correct}}) - \overline{\cos\text{sim}(\Delta h,\, v_{\text{wrong}})}$$

Hypothesis prediction: $\text{gap} > 0$, most pronounced in the key layers.

2.4 The Nature of Reasoning: Iterative Refinement on the Residual Stream

🎯 Core claim: Generating each reasoning token = one attention computation = one update to the hidden state on the residual stream = one step closer toward the key associated with the correct answer direction.

Chain-of-Thought (CoT) does not make the model "think" — it executes additional projection steps on the residual stream, each moving the hidden state closer to the correct answer's subspace, until the EOS token's logit crosses a threshold. This explains why "think step by step" or "think longer" improves accuracy: it gives the residual stream more chances to converge.

2.5 Critical Token and Saddle Point

Existing literature defines critical tokens as "tokens with the highest contribution to the final answer," identified post-hoc via attention or gradient attribution. This captures correlation, not causation. This hypothesis proposes a revised definition:

💡 Revised definition: A critical token is the token at position $t+1$ after a saddle point — it determines which attractor basin the residual stream trajectory enters, and thereby determines where the entire reasoning chain converges.

A saddle point is where the curvature of the residual stream trajectory is near zero: the system is maximally sensitive to the direction taken at the next step. Geometric analogy: standing at the top of a saddle, you slide left into one valley or right into another. The critical token is the one that decides left or right.

This explains three phenomena: (1) CoT chains that begin with a wrong step are almost always entirely wrong — the trajectory entered the wrong basin; (2) Self-consistency works — multiple samples near the saddle point, with majority voting against local minima; (3) A false premise in the prompt corrupts the entire answer — the trajectory is pushed into the wrong basin before even reaching the saddle point.

2.6 The Geometric Effect of RL: $x(x-3) \to 5x(x-3)$

What does reinforcement learning actually do to the model? An analogy:

$$\text{Before RL: } f(x) = x(x-3) \quad \text{(zeros at } x=0 \text{ and } x=3\text{; capability boundary unchanged)}$$ $$\text{After RL: } f(x) = 5x(x-3) \quad \text{(zeros unchanged; gradient magnitude amplified 5×)}$$

What RL does
Amplifies the gradient magnitude of the loss landscape. Makes trajectories near saddle points more stable. Makes it more likely that critical tokens select the correct direction.

What RL does not do
It does not move the zeros (capability boundary unchanged). The base model's semantic manifold already determines the upper bound of capability — RL cannot exceed it, only reach it more reliably.

💡 Capability ceiling hypothesis: For any fixed context, the correct answer token for position $n+1$ always has a nonzero softmax probability — it is always present in the logit list. RL only shifts its probability from $p$ to $p' > p$. The capability boundary is set by the base model's semantic manifold; RL cannot create new manifold, only pave roads to what already exists. RL and Best-of-N are equivalent operations at the single-step level — one amortized at training time, one paid at inference time.

Over-optimization: when the landscape becomes too steep, the model loses its ability to explore near saddle points — the mechanism behind reward hacking and mode collapse.

III. Exp A: Geometric Directionality of the MLP Guider

3.1 Research Question

Does the MLP output vector $\Delta h$ geometrically point toward the correct answer's direction?

3.2 Experimental Design

Model: Qwen2.5-3B. Dataset: TriviaQA (factual QA with unambiguous correct answers). For each sample and each layer:

Δh[layer] = MLP_output[layer][-1, :] # last token position only v_correct = lm_head.weight[correct_token_id] v_wrong_k = lm_head.weight[wrong_token_id_k] # 10 random wrong answers gap = cos_sim(Δh, v_correct) − mean_k(cos_sim(Δh, v_wrong_k))

Key metrics: $\text{gap} > 0$ and positive-gap sample rate $> 50\%$. The hypothesis predicts both hold in the key layers with significant deviation from the null.

3.4 Key Findings

The original prediction was "deeper layers have larger gap, last four layers strongest." The actual result reveals a clear Guider peak at L31–32 followed by reversal. The gap of 0.033 is modest in cosine similarity terms, but 80% of samples show a positive gap — against a null expectation of 50%, the deviation exceeds 6 standard deviations. This is not noise.

The mildly negative gap in the middle layers (L4–29) is not a counterexample: those layers are performing semantic understanding, which requires temporarily moving away from the answer direction — a necessary part of the model's division of labor. The terminal reversal (L33–35) is explained by Exp-LL.

IV. Exp-LL: Logit Lens — Two-Mechanism Separation

4.1 Research Question

Exp A found that $\Delta h$ reverses direction in L33–35, yet $p_{\text{correct}}$ should continue improving. How do we resolve this contradiction?

4.2 Experimental Design (Logit Lens)

Project each layer's hidden state through lm_head and record three metrics:

  • $p_{\text{correct}}$: Post-softmax probability of the correct answer token. The probability of selecting the correct answer if the model were to output at this layer.
  • $\text{logit\_gap}$: The correct answer's logit minus the mean logit across all tokens. Measures how prominent the correct answer is in logit space.
  • rank: The rank of the correct answer token by probability. rank = 1 is best; lower is always better.

4.4 Core Finding: The Two Peaks Do Not Coincide

🔑 Key observation: logit_gap peaks at L32 (11.91) then declines. $p_{\text{correct}}$ and rank continue improving after L32, only reaching their best at L35. This means what happens after L32 is not continued pushing in the correct-answer direction — it is compressing the overall distribution to be sharper, eliminating probability mass from other tokens and amplifying the correct answer's relative advantage.

Two geometrically distinct operations are revealed:

Operation 1: Direction Setting — L31–32
$\Delta h$ strongly aligned with $v_{\text{correct}}$. Rotates the residual stream's main direction toward the correct answer's subspace. logit_gap peaks at 11.91. "I know which direction the answer lies."

Operation 2: Distribution Sharpening — L33–35
$\Delta h$ direction reverses, but eliminates probability mass from competing candidates. rank drops from 3,909 to 155; $p_{\text{correct}}$ rises from 0.135 to 0.207. "I am making this answer unambiguous."

✅ Resolving the Exp A contradiction: The terminal reversal in Exp A (negative gap at L33–35) does not falsify the hypothesis. L33–35 MLPs are performing Distribution Sharpening, not Direction Setting — their $\Delta h$ does not point toward the correct answer, but toward "the direction that eliminates other candidates." This is geometrically opposite, yet causes $p_{\text{correct}}$ to continue rising.

Analogy: Direction Setting is pointing the telescope at the target; Distribution Sharpening is adjusting the focus until the image is sharp. Both are necessary, but they are entirely different geometric operations occurring in different layers.

Together they determine the final $P(\text{correct token})$.

VI. Failure Modes

  • Hallucination: The correct answer's key is absent from context. The residual stream cannot reach the correct attractor basin and converges to a local minimum that is geometrically stable but semantically wrong. The model has not failed to converge — it has converged to the wrong place.
  • Confidently wrong CoT: The critical token after the saddle point selects the wrong direction, entering the wrong attractor basin. All subsequent reasoning tokens continue converging inside the wrong basin, reinforcing an incorrect direction — while the model remains highly confident.
  • Over-RL: The landscape becomes too steep ($x(x-3) \to 1000x(x-3)$). Saddle points become so sharp that the model loses its ability to explore near them, always taking the same path. This is the mechanism behind reward hacking, mode collapse, and catastrophic failure on out-of-distribution inputs.

VII. Open Questions

  1. Exp-SP (Saddle Point Confirmation): Is the L28–29 gap trough a genuine saddle point? Perturb hidden states at L28–29 with small noise and measure whether the final output changes more than at other layers (perturbation sensitivity test). If confirmed, critical tokens most likely appear at L30 and beyond.
  2. Exp-BYP (Bypass L33–35): If L33–35 is skipped and the L32 hidden state is fed directly to lm_head, does accuracy rise or fall? Prediction: accuracy drops, rank distribution broadens — confirming Distribution Sharpening is not optional.
  3. Exp B (MoE Router Ablation): Requires a MoE model (e.g., Qwen1.5-MoE-A2.7B). Prediction: forcing the deactivation of router-selected experts degrades performance far more than deactivating an equal number of randomly chosen experts.

⚠️ Claims requiring formalization: (1) The "capability ceiling" is currently defined as "the boundary of the base model's semantic manifold" — this boundary is not directly measurable in practice, making the claim technically unfalsifiable without a more operational definition. (2) The saddle point is currently approximated by the entropy proxy (position of peak output distribution entropy); Exp-SP aims to establish a more rigorous geometric definition.

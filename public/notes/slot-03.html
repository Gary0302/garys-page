SNN N! Ordering Space

Spiking Neural Network（脈衝神經網路）中，M 個 neuron 在 T 個時間步內的發放順序，構成一個 M! 大小的排列空間。目前沒有任何研究成功將這個空間作為主要資訊載體加以利用。

核心主張

在 4 個 neuron 的 SNN 中，spike ordering「1423」與「4231」攜帶完全不同的資訊；而「1423」與「1432」因為前兩個 spike 相同，應在語意空間中非常靠近。這個距離結構，是這個框架想要學習並利用的。

本框架的設計目標是：讓 SNN 的訓練信號直接來自 ordering 結構，而非 firing rate 或 spike count 等傳統指標，從而真正開發 N! 空間的表達能力。

核心定義

SNN（Spiking Neural Network）

脈衝神經網路。與傳統 ANN 不同，SNN 的 neuron 只在膜電位（membrane potential）超過閾值時發出離散的「spike」（脈衝），其餘時間保持靜默。資訊以 spike 的時間而非持續的激活值來編碼。

LIF（Leaky Integrate-and-Fire）

最常見的 SNN neuron 模型。膜電位以速率 β（beta）衰減（leaky），並隨輸入電流積分（integrate）。當電位超過閾值時發放 spike，之後重置。

mem[t] = β · mem[t-1] + input[t] ; spk = 1 if mem > threshold
State Leaky（狀態疊加漏積分）

兩層串聯的 LIF neuron：第一層（β₁=0.9）的 spike 作為第二層（β₂=0.95）的輸入。這使得第二層的 spike 更平滑、更分散在時間軸上，讓 ordering 訊號更易於被學習，避免所有 neuron 在同一時間步集中發放。

spk1, mem1 = lif1(x[t], mem1) ; spk2, mem2 = lif2(spk1, mem2)
Spike Matrix S

Pass 1 的輸出。一個 M×T 的二值矩陣，每一行代表一個 neuron 在 T 個時間步中的 spike 記錄（0 或 1）。這是 ordering pattern 的原始表示，不壓縮、不可微分化。

S ∈ {0,1}^(M×T) ; S[i,t] = 1 表示 neuron i 在時間步 t 發放
Spike Ordering（發放排列）

從 Spike Matrix 中讀取每個 neuron 第一次發放的時間步，形成的排列。M 個 neuron 的所有可能排列共有 M! 種，構成本框架的核心表示空間。

ordering = argsort([first_spike_time(n) for n in neurons])
RPC（Recursive Proportional Cutting，遞迴比例切割）

將 spike ordering 映射到 [0,1] 區間上的座標系的方法。第一個 spike 的時間決定第一刀的切割比例，後續每個 spike 在剩餘子區間中繼續切割，遞迴進行。幾何上對應 M! 個頂點的 Ordering Simplex 中的一條路徑。這是一個固定的、無參數的映射。

coord(ordering) → [0,1]^(M-1) ; 相似排列 → 座標近，不同排列 → 座標遠
Time-Weighted Loss（時間加權損失）

對不同時間步的預測錯誤給予不同懲罰的損失函數。越早的 spike 的預測錯誤，懲罰越高（因為第一個 spike 攜帶最多資訊）。越晚的 spike 的錯誤，懲罰指數衰減至接近 0。

w(t) = exp(-α·t) ; Loss = Σ_t w(t) · CE(ŷ_t, y_t)
N! Ordering Space（N! 排列空間）

M 個 neuron 所有可能發放排列構成的空間，大小為 M!。本框架的核心假設是：這個空間擁有豐富的幾何結構（距離、相似性、拓撲），可以作為語意空間使用，且目前尚未有研究成功加以利用。

|空間| = M! ; e.g. M=8 → 40,320 個獨立的排列座標

State Leaky 的設計

單層 LIF 的 spike 過於尖銳，容易導致所有 neuron 在同一時間步集中發放，使得 ordering 無法被有效區分。State Leaky 透過讓第一層的 spike 作為第二層的輸入，增加了時間延遲與平滑性：

python state_leaky_forward 
# 初始化
lif1 = snn.Leaky(beta=0.9)
lif2 = snn.Leaky(beta=0.95)
mem1 = lif1.init_leaky()
mem2 = lif2.init_leaky()

spk2_rec = []

for t in range(T):
    cur = self.fc_in(x)              # (batch, M)
    spk1, mem1 = lif1(cur,  mem1)    # 第一層：產生初始 spike
    spk2, mem2 = lif2(spk1, mem2)    # 第二層：平滑化，延遲 spike
    spk2_rec.append(spk2)

# S ∈ {0,1}^(batch, M, T)
S = torch.stack(spk2_rec, dim=-1)
為什麼 β₂ > β₁

β 越大，膜電位衰減越慢，積分時間常數越長。LIF₂ 的 β=0.95 讓它對輸入更敏感但更「懶」於發放，天然地拉開了不同 neuron 第一次發放的時間間距，使得 ordering 的資訊量更豐富。

Spike Matrix 的表示

S 是一個 M×T 的二值矩陣。每一行是一個 neuron 的時序 spike 記錄。我們不壓縮這個矩陣，不對它做 argmax 或 softmax，直接保留完整資訊。

以 4 個 neuron、6 個時間步為例：

Neuron t=0t=1t=2t=3t=4t=5 First spike at
N₁ 1 00 100 t=0 → 排列位置 1
N₂ 00 1 001 t=2 → 排列位置 3
N₃ 0 1 0001 t=1 → 排列位置 2
N₄ 000 1 10 t=3 → 排列位置 4

這個 Spike Matrix 對應的 ordering 是：1 3 2 4（按 first spike time 排序的 neuron 序號）。

遞迴比例切割（RPC）

RPC 是一個固定映射無參數，將 spike ordering 轉換為連續 embedding 空間中的座標。核心保證：幾何距離 = 語意距離。

切割規則

從區間 [0, 1] 開始，每個 spike 的發放時間決定在當前剩餘區間內的切割比例：

├─ N1@t=0 → cut: 0/4=0.00

RPC 的幾何性質

RPC 建構的座標系天然實現了「越早的 spike 貢獻越大的距離」。1423 和 1432 在前兩刀（佔總距離的前 50%+）上完全相同，所以座標很近。4231 在第一刀就分叉，距離最遠。這與人類直覺對「相似排列」的理解完全吻合。

RPC 的數學形式

給定 ordering π = [π₁, π₂, ..., π_M]，first_spike_time = [t₁, t₂, ..., t_M]

coord₁ = t_{π₁} / T ← 第一個 spike 的時間比例
coord₂ = t_{π₂} / (T - t_{π₁}) ← 在剩餘時間中的比例
coord₃ = t_{π₃} / (T - t_{π₂}) ← 遞迴進行
...

→ embedding = [coord₁, coord₂, ..., coord_{M-1}] ∈ [0,1]^(M-1)
python rpc_embedding（固定函數，無參數） 
def rpc_embedding(spike_matrix: torch.Tensor, T: int) -> torch.Tensor:
    """
    spike_matrix: (batch, M, T)  — Spike Matrix S
    回傳:          (batch, M-1)   — RPC 座標向量
    """
    batch, M, _ = spike_matrix.shape

    # 取每個 neuron 的 first spike time（未發放的 neuron 給 T）
    has_spike = (spike_matrix > 0)
    first_spike = torch.where(
        has_spike.any(dim=-1),
        has_spike.float().argmax(dim=-1),
        torch.full((batch, M), T, dtype=torch.float)
    )  # (batch, M)

    # 排序以得到 ordering（不影響梯度流，RPC 本身不需要）
    ordering = first_spike.argsort(dim=-1)  # (batch, M)

    # 按 ordering 重排 first_spike_time
    sorted_times = torch.gather(first_spike, 1, ordering)  # (batch, M)

    # 遞迴切割：計算每個 spike 在剩餘時間中的比例
    coords = []
    remaining = torch.full((batch,), T, dtype=torch.float)

    for i in range(M - 1):
        t_i = sorted_times[:, i]
        coord_i = t_i / (remaining + 1e-8)
        coords.append(coord_i.clamp(0, 1))
        remaining = remaining - t_i

    return torch.stack(coords, dim=-1)  # (batch, M-1)

Pass 2 — 讀出層

Pass 2 負責從 RPC 座標向量讀出最終預測。本框架刻意最小化 Pass 2 的參數數量，確保學習功勞歸屬清晰。

設計哲學：最小化可學習變因

W1, W2 若作為可學習矩陣，有兩個根本問題：(1) 線性映射無法保留排列空間的拓撲幾何；(2) Pass 2 可能繞過 ordering，直接從 spike density 等 spurious feature 學到答案，導致無法確認是 ordering 在起作用。RPC 是固定映射，所有歸納能力必須來自 Pass 1 的 SNN。

Pass 2 的選項

方案參數量描述建議階段
A 固定相似度量 0 直接計算 RPC 座標間的歐氏距離或餘弦相似度，對應到 class label Exp 1 — 最純粹
B 單層線性讀出 少量 nn.Linear(M-1, C)，輸入 RPC 座標，輸出 class logits Exp 2 — 主方案
C 固定分群中心 0 每個 class 的 RPC 座標聚類中心，以最近鄰分類 Exp 3 — 分析用
python pass2_readout（方案 B） 
# rpc_coord: (batch, M-1)  — 來自固定 RPC 函數
self.readout = nn.Linear(M - 1, C)   # C = 類別數

def forward(self, spike_matrix):
    rpc_coord = rpc_embedding(spike_matrix, self.T)  # 固定函數
    logits = self.readout(rpc_coord)                  # 唯一可學習部分
    return logits

Loss 設計

損失函數由兩個部分組成：標準分類 cross-entropy，以及時間加權懲罰（越早的 spike 預測錯誤懲罰越重）。

時間加權的直覺

Ordering 的語意主要由前幾個 spike 決定（類似於 RPC 中越早的切割權重越高）。若第一個 spike 預測錯了，代表整個 ordering 的方向就錯了。因此：

懲罰權重 w(t) = exp(-α·t)，α = 0.5 時：
L_total = L_CE(y, ŷ) + λ · L_tw

L_tw = Σt=0T-1 exp(-α·t) · CE(ŷ_t, y)

# 其中 ŷ_t 是在時間步 t 的 spike 預測，y 是 class label
# 如果 neuron 從未發放（no spike），懲罰為 0（已錯失機會）
python time_weighted_loss 
def time_weighted_loss(
    logits_per_t: torch.Tensor,  # (T, batch, C)
    target:       torch.Tensor,  # (batch,)
    alpha: float = 0.5,
    lam:   float = 1.0
) -> torch.Tensor:
    T = logits_per_t.shape[0]
    t_idx = torch.arange(T, dtype=torch.float, device=target.device)

    # 指數衰減權重，歸一化
    weights = torch.exp(-alpha * t_idx)
    weights = weights / weights.sum()

    # 加權求和
    loss_tw = torch.stack([
        F.cross_entropy(logits_per_t[t], target) * weights[t]
        for t in range(T)
    ]).sum()

    return loss_tw * lam

設計邏輯

這個框架的每一個選擇都對應一個明確的實驗邏輯。以下是核心決策與其理由：

決策選擇理由
Pass 1 輸出 直接 Spike Matrix，不壓縮 保留完整 ordering 資訊；不引入 argmax/softmax 等不可微或失真的操作
Pass 2 映射 RPC（固定函數） 消除 W1/W2 作為可學習矩陣帶來的 spurious feature 風險；幾何距離直接對應語意距離
Neuron 模型 State Leaky（雙層 LIF） 拉開不同 neuron 的發放時間間距，讓 ordering 的資訊量最大化
Loss 設計 Time-weighted CE 讓訓練信號對應 ordering 的幾何重要性：越早的 spike 錯誤 → 越大的梯度
變因控制 最小化 Pass 2 參數 確保實驗結論歸因清晰：如果學得好，是 SNN 學會了 ordering，不是 Pass 2 補救了

實驗計畫

Exp 1
Sanity Check：ordering 可分性
在 toy dataset 上驗證 SNN 是否能學到不同 class 對應不同 ordering。Pass 2 用固定相似度量（零參數）。
Exp 2
RPC vs Linear Readout
比較 RPC 固定映射 vs 可學習 W1 的性能。確認 ordering 幾何是否真的被利用，而非被 W 補救。
Exp 3
M 的 scaling
測試 M=4（24種排列）到 M=8（40,320種排列）的性能曲線，確認 N! 空間的 scaling 是否有效。
Exp 4
Ordering 幾何可視化
用 t-SNE 將 RPC 座標投影到 2D，確認不同 class 的 ordering 在空間中是否形成可分的群集。

完整框架程式碼

python snn_ordering_framework.py 
import torch
import torch.nn as nn
import torch.nn.functional as F
import snntorch as snn

# ─────────────────────────────────────────
# 固定函數（無參數）：Spike Matrix → RPC 座標
# ─────────────────────────────────────────

def rpc_embedding(S: torch.Tensor, T: int) -> torch.Tensor:
    """
    S    : (batch, M, T)  — Spike Matrix，來自 Pass 1
    T    : int            — 時間步數
    回傳  : (batch, M-1)   — RPC 連續座標向量
    """
    batch, M, _ = S.shape

    # 計算每個 neuron 的 first spike time
    has_spike  = (S > 0)
    first_time = torch.where(
        has_spike.any(dim=-1),
        has_spike.float().argmax(dim=-1),
        torch.full((batch, M), float(T))
    )  # (batch, M)

    # 按 first spike time 排序，得到 ordering
    ordering     = first_time.argsort(dim=-1)       # (batch, M)
    sorted_times = torch.gather(first_time, 1, ordering)  # (batch, M)

    # 遞迴切割：每個 spike 在剩餘時間中的比例
    coords    = []
    remaining = torch.full((batch,), float(T), device=S.device)

    for i in range(M - 1):
        t_i     = sorted_times[:, i]
        coord_i = (t_i / (remaining + 1e-8)).clamp(0, 1)
        coords.append(coord_i)
        remaining = (remaining - t_i).clamp(min=0)

    return torch.stack(coords, dim=-1)  # (batch, M-1)


# ─────────────────────────────────────────
# Time-Weighted Loss
# ─────────────────────────────────────────

def time_weighted_loss(
    logits_t: torch.Tensor,  # (T, batch, C)
    target:   torch.Tensor,  # (batch,)
    alpha: float = 0.5,
    lam:   float = 1.0
) -> torch.Tensor:
    T  = logits_t.shape[0]
    w  = torch.exp(-alpha * torch.arange(T, dtype=torch.float, device=target.device))
    w  = w / w.sum()
    losses = torch.stack([F.cross_entropy(logits_t[t], target) * w[t] for t in range(T)])
    return losses.sum() * lam


# ─────────────────────────────────────────
# 主模型
# ─────────────────────────────────────────

class SNNOrderingFramework(nn.Module):
    def __init__(self, N: int, M: int, C: int, T: int,
                 beta1=0.9, beta2=0.95):
        """
        N : 輸入維度
        M : SNN neuron 數量（ordering 的 M in M!）
        C : 類別數
        T : 時間步數
        """
        super().__init__()
        self.T = T
        self.M = M

        # Pass 1：輸入投影 + State Leaky
        self.fc_in = nn.Linear(N, M)
        self.lif1  = snn.Leaky(beta=beta1)
        self.lif2  = snn.Leaky(beta=beta2)

        # Pass 2：單層線性讀出（唯一可學習部分）
        self.readout = nn.Linear(M - 1, C)

    def forward(self, x: torch.Tensor):
        """
        x : (batch, N)
        回傳 : logits (batch, C)，spike_matrix (batch, M, T)
        """
        batch = x.shape[0]
        mem1  = self.lif1.init_leaky()
        mem2  = self.lif2.init_leaky()
        spk_rec = []

        # ── Pass 1：SNN 前向，收集 Spike Matrix ──
        for _ in range(self.T):
            cur         = self.fc_in(x)
            spk1, mem1  = self.lif1(cur,  mem1)
            spk2, mem2  = self.lif2(spk1, mem2)
            spk_rec.append(spk2)

        S = torch.stack(spk_rec, dim=-1)  # (batch, M, T)

        # ── RPC（固定函數）→ 連續 ordering 座標 ──
        rpc_coord = rpc_embedding(S, self.T)  # (batch, M-1)

        # ── Pass 2：線性讀出 ──
        logits = self.readout(rpc_coord)  # (batch, C)

        return logits, S


# ─────────────────────────────────────────
# 訓練迴圈（示意）
# ─────────────────────────────────────────

def train_step(model, x, y, optimizer, alpha=0.5, lam=1.0):
    optimizer.zero_grad()
    logits, S = model(x)

    # 主 loss：分類
    loss_ce = F.cross_entropy(logits, y)

    # 輔助：時間加權（需要 per-timestep logits，此處示意）
    # 實作時可在 forward 中額外儲存 per-t 的 readout logits

    loss = loss_ce  # + time_weighted_loss(logits_t, y, alpha, lam)
    loss.backward()
    optimizer.step()
    return loss.item()
